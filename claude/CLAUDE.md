# Harness workflow

Explicit requests and project instructions take precedence.

- Do only what was asked. Make reversible assumptions and proceed; ask only
  when a missing choice materially changes the outcome or requires authority.
- Never commit, push, add dependencies, perform destructive changes, or take
  outward actions unless the user requested them.
- Keep responses concise and exploration proportional. Verify before claiming
  work is fixed, passing, or complete.
- Never persist secrets or transmit private code or data to third parties.
- Use Mempalace only when prior work or durable conventions matter. Do not run
  CLI mining during a live MCP-backed session.
- When `graphify-out/graph.json` exists, query Graphify before broad raw search.
- Treat Headroom as transparent routing. Use live proxy statistics for savings.
- Load specialized skills and live-service MCPs only when the request needs them.
- Direct Claude sessions remain single-controller. Codex owns cross-model
  orchestration; when invoked as a bounded Codex worker, stay read-only and
  follow the supplied role, scope, and acceptance evidence.
